master_builder module¶

This script builds the Bumps application and documentation from source and runs unit tests and doc tests. It supports building on Windows and Linux.

Usually, you downloaded this script into a top-level directory (the root) and run it from there which downloads the files from the application repository into a subdirectory (the package directory). For example if test1 is the root directory, we might have:

E:/work/test1/master_builder.py

/bumps/master_builder.py /bumps/…

Alternatively, you can download the whole application repository and run this script from the application’s package directory where it is stored. The script determines whether it is executing from the root or the package directory and makes the necessary adjustments. In this case, the root directory is defined as one-level-up and the repository is not downloaded (as it is assumed to be fully present). In the example below test1 is the implicit root (i.e. top-level) directory.

E:/work/test1/bumps/master_builder.py

/bumps/…

Need to update this for bumps rather refl1d standalone gui distribution.
